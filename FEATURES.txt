Feature Summary
==============================

Main App Pages
--------------
- Home page: Main landing dashboard with shortcuts to equipment rental, escort services, assistance, bookings, feedback, login, and registration.
- Explore / Equipment page: Lets users browse equipment and care services, search listings, filter by stock or schedule, sort by price, and open item details.
- Services / Escort dashboard page: Shows escort requests and volunteer availability slots based on the user's role, with status tracking for pending, matched, confirmed, and completed jobs.
- Delivery page: Shows active equipment orders and service bookings in a searchable, filterable timeline with latest order or booking details.
- Settings page: Lets users change language, accessibility theme presets, font size controls, advanced theme colors, and view app version information.
- Profile page: Shows the signed-in user's account details, role, admin entry button, settings/help links, certifications, password change, and logout.

Authentication Pages
--------------------
- Register page: Creates a new account and lets users choose whether they are signing up as a Senior or Escort volunteer.
- Login page: Signs existing users into their TwinXCare account.
- Change Password page: Lets signed-in users securely update their password.
- Logout page: Handles sign-out behavior.

Equipment Rental Pages
----------------------
- Equipment detail / Order page: Shows the selected equipment item, price, description, stock, pickup warehouse options, rental quantity, rental dates, and swipe-to-rent action.
- Payment page: Confirms equipment rentals or service bookings, shows the total cost, collects delivery address for equipment orders, updates stock or service capacity, saves the transaction, and sends notifications.
- My Bookings page: Shows confirmed service bookings with service name, date, time, and details.

Escort Pages
------------
- Request Escort page: Lets senior/patient users request medical escort help by entering appointment time, hospital, pickup location, contact details, medical notes, and required certification.
- Volunteer Availability page: Lets escort volunteers post availability with date, time range, location, travel radius, and contact phone.
- Escort Assignment page: Shows matched escort job details, patient or volunteer information, confirmation controls, completion controls, and rating flow.
- Certifications page: Lets escort volunteers submit certification links for admin review and view submission status.

Support Pages
-------------
- Assistance page: Lets users submit help requests to the support team.
- Feedback page: Lets users submit comments and a star rating.
- Help Docs page: Provides basic guidance for using the app, navigation, password changes, support, and escort mode.

Admin Pages
-----------
- Admin Control Center page: Main admin menu linking to inventory, services, deliveries, users, escort matching, certifications, and inbox tools.
- Admin Equipment Management page: Lets admins add, edit, search, and delete equipment, manage warehouse pickup points, and control stock per warehouse.
- Admin Service Management page: Lets admins add and remove care services, manage service details, and create or delete service schedule slots.
- Admin Delivery Management page: Lets admins view user orders, search by transaction ID, update delivery ETA, and mark orders complete or incomplete.
- Admin User Management page: Lets admins view registered users, change roles, and delete non-admin accounts through the backend.
- Admin Escort Matching page: Lets admins view pending escort requests and available volunteers, send compromise notifications, or force a manual match.
- Admin Certifications page: Lets admins manage certification types and approve or revoke volunteer certification submissions.
- Admin Support Inbox page: Lets admins review feedback and assistance requests, search/filter entries, archive/unarchive, and delete entries.

System and Backend Features
---------------------------
- Automatic escort matching: Matches patient requests with volunteer availability using schedule, location/radius, certification, and scoring rules.
- Notifications: Uses local and Expo push notifications for orders, bookings, escort matches, confirmations, completions, and admin requests.
- Remote version check: Blocks outdated app versions until users update.
- Firebase backend: Uses Firebase Auth and Firestore for accounts, equipment, services, bookings, escort workflows, feedback, assistance requests, and admin data.
- Static web pages: Includes public web pages and app preview pages under the public folder.
